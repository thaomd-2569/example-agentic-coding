II. DESCRIPTION
────────────────────────────────────────────────────────────────────────
This feature allows users to export the current (filtered or full) transaction list to a CSV file with columns: Date, Type, Category, Amount (VND), Note. The filename follows the format transactions_YYYY-MM-DD.csv. Export is performed client-side using the Blob API, with a toast notification on success. If there are no transactions, the Export button is disabled and a tooltip explains why.

V. PLAN SUMMARY
────────────────────────────────────────────────────────────────────────
Architecture Decisions:
  • CSV Generation: Manual string build — Simple, no dependencies, full control
  • Download Method: Blob API + anchor click — Native, no backend, works in all browsers
  • Filename Format: transactions_YYYY-MM-DD.csv — Matches spec, user-friendly
  • Notification: Toast (existing/toast lib) — Consistent UX, instant feedback
  • Button State: Disabled + Tooltip — Prevents errors, clear UX

Tech Stack:
  UI      : React 18+, Tailwind
  State   : Zustand
  Utils   : Custom CSV util
  Notification : Toast lib (react-hot-toast)
